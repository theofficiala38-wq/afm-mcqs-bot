# 🤖 AFM MCQs BOT — دليل إعداد GitHub Actions

## الهيكل المطلوب في المستودع

```
your-repo/
├── index.html                          ← الملف الرئيسي (يُنشئه الأدمن من التطبيق)
├── .nojekyll                           ← يمنع Jekyll من معالجة الملفات
├── .github/
│   └── workflows/
│       ├── deploy.yml                  ← النشر التلقائي على GitHub Pages
│       ├── validate-pr.yml             ← فحص Pull Requests
│       └── auto-publish-check.yml      ← إشعار عند رفع الأدمن مباشرة
└── SETUP.md                            ← هذا الملف
```

---

## ⚙️ خطوات الإعداد الأولي (مرة واحدة فقط)

### 1. تفعيل GitHub Pages

1. افتح **Settings** في المستودع
2. انتقل إلى **Pages** (من القائمة الجانبية)
3. في **Source**، اختر: **GitHub Actions**
4. احفظ الإعدادات

> ⚠️ مهم: اختر **"GitHub Actions"** وليس "Deploy from a branch"

---

### 2. رفع الملفات الأولية

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO

# انسخ الملفات من هذا المستودع
cp index.html .nojekyll .github -r /path/to/your-repo/

git add .
git commit -m "🚀 Initial AFM MCQs BOT setup"
git push origin main
```

---

### 3. التحقق من النشر

بعد الـ push، افتح **Actions** في GitHub:
- يجب أن يظهر workflow اسمه "🚀 Deploy AFM MCQs BOT"
- انتظر حتى يكتمل (عادة 1-2 دقيقة)
- الرابط النهائي: `https://YOUR_USERNAME.github.io/YOUR_REPO/`

---

## 🔄 كيف تعمل التحديثات؟

### طريقة 1: النشر عبر التطبيق مباشرة (الأسرع) ⭐

```
الأدمن يضيف أسئلة
    ↓
يضغط "⚙️ إعدادات GitHub" ← يُعيّن التوكن مرة واحدة فقط
    ↓
يضغط "نشر على GitHub"
    ↓
GitHub Actions يتحقق ← يُنشر تلقائياً (30-60 ثانية)
    ↓
كل الطلاب يحصلون على الأسئلة الجديدة فور فتح التطبيق ✅
```

### طريقة 2: التصدير اليدوي ثم الرفع

```
الأدمن → "📥 تصدير الموقع" → يُنزَّل index.html جديد
    ↓
يرفع الملف على GitHub (drag & drop في واجهة الويب)
    ↓
GitHub Actions يُنشر تلقائياً
    ↓
كل الطلاب يحصلون على التحديث ✅
```

## 🔑 إعداد توكن GitHub للأدمن

### إنشاء التوكن (مرة واحدة)

1. افتح: https://github.com/settings/tokens/new
2. **Note**: `AFM MCQs BOT`
3. **Expiration**: 90 days (أو No expiration)
4. **Scopes**: ✅ `repo` (كل الصلاحيات الفرعية)
5. اضغط **Generate token**
6. **انسخ التوكن فوراً** (لن يظهر مرة أخرى)

### إدخال التوكن في التطبيق

1. افتح التطبيق على المتصفح
2. أدخل كلمة سر الأدمن
3. افتح **⚙️ إعدادات GitHub**
4. أدخل:
   - **التوكن**: ghp_xxxxxxxxxxxx
   - **المالك (Owner)**: اسم حسابك على GitHub
   - **المستودع (Repo)**: اسم المستودع
   - **المسار (Path)**: `index.html`
   - **الـ Branch**: `main`
5. اضغط **حفظ الإعدادات**

> 🔒 **الأمان**: التوكن لا يُحفظ في الملف أبداً — يعيش مشفراً في متصفح الأدمن فقط

## 🔁 آلية وصول التحديثات للطلاب

```
التطبيق يخزن APP_VERSION في localStorage
         ↓
كل مرة يفتح الطالب التطبيق:
         ↓
يقارن APP_VERSION المخزون مع APP_VERSION في الملف الجديد
         ↓
إذا اختلفا → location.reload(true) تلقائي
         ↓
الطالب يرى الأسئلة الجديدة فوراً ✅ (بدون أي تدخل)
```

**النتيجة**: بمجرد نشر الأدمن التحديث، كل طالب يفتح التطبيق سيحصل على الإصدار الجديد تلقائياً.

---

## 📊 Workflows الموجودة وأدوارها

| Workflow | متى يُشغَّل | الهدف |
|---------|------------|-------|
| `deploy.yml` | عند كل push لـ `index.html` | النشر الكامل على GitHub Pages |
| `validate-pr.yml` | عند فتح Pull Request | فحص الملف قبل الدمج |
| `auto-publish-check.yml` | عند كل push لـ `index.html` | فحص سريع + ملخص عبر Actions |

---

## ❓ أسئلة شائعة

**س: هل يحتاج الطلاب لتحديث يدوي؟**
ج: لا، التطبيق يتحقق تلقائياً من `APP_VERSION` عند كل فتح للصفحة.

**س: كم يستغرق وصول التحديث للطلاب؟**
ج: عادةً 1-3 دقائق بعد نشر الأدمن، ثم أول فتح للتطبيق.

**س: ماذا لو كان الطالب يستخدم التطبيق أثناء النشر؟**
ج: في المرة القادمة التي يفتح فيها التطبيق سيحصل على التحديث.

**س: هل يُحذف تقدم الطالب عند التحديث؟**
ج: يُحفظ تقدم الطالب في `localStorage` إلى أن ينتهي session الأسئلة. عند التحديث، تُحفظ إعدادات الثيم وإعدادات GitHub وتُمسح بقية البيانات المؤقتة.

**س: هل يمكن استخدام Custom Domain؟**
ج: نعم، أضف ملف `CNAME` يحتوي اسم الدومين، وأعدّ DNS. التطبيق يعمل على أي دومين.

## 🚨 استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| Actions يفشل بخطأ permissions | تأكد من تفعيل GitHub Pages بـ "GitHub Actions" كـ source |
| الطلاب لا يرون التحديث | تحقق من أن APP_VERSION تغيّر فعلاً في الملف الجديد |
| خطأ 401 عند النشر عبر التطبيق | التوكن منتهي أو خاطئ — جدّده من GitHub Settings |
| خطأ 403 عند النشر | التوكن لا يملك صلاحية `repo` — أنشئ توكناً جديداً |
| الصفحة تُعرض فارغة | تأكد من وجود `.nojekyll` في جذر المستودع |
