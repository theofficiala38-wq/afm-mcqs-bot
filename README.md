# 🔐 نشر آمن على GitHub Pages

## المشكلة
GitHub بيكتشف التوكن في الكود ويوقفه تلقائياً (GitHub Secret Scanning).

## الحل
ملف GitHub Actions ينشر **نسخة نظيفة** من `index.html` على GitHub Pages:
- التوكن **لا يُكتب في أي كود** أبداً
- قبل النشر، يُمسح `GH_ENCRYPTED` تلقائياً من الملف المنشور
- يُنشر فقط `index.html` المنظّف — لا شيء آخر من الريبوزيتوري
- يعمل بـ `GITHUB_TOKEN` الأوتوماتيكي — **لا إعداد يدوي للتوكن**

---

## خطوات الإعداد (مرة واحدة فقط)

### 1. انسخ مجلد .github إلى رووت الريبوزيتوري
```
your-repo/
├── index.html
└── .github/
    └── workflows/
        └── deploy.yml    ← هذا الملف
```

### 2. فعّل GitHub Pages من فرع gh-pages
في الريبوزيتوري:
- **Settings → Pages**
- **Source:** Deploy from a branch
- **Branch:** `gh-pages` → `/ (root)`
- اضغط **Save**

### 3. خلاص! لا يوجد إعداد إضافي للتوكن
`secrets.GITHUB_TOKEN` متاح تلقائياً في كل ريبوزيتوري.

---

## طريقة الاستخدام

### النشر التلقائي
كل ما تعمل `push` على `main` أو `master`:
1. الـ workflow بيشتغل تلقائياً
2. يمسح `GH_ENCRYPTED` من الملف
3. ينشر النسخة النظيفة على `gh-pages`

### النشر اليدوي
1. **Actions** في ريبوزيتوريك
2. اختر **"نشر الموقع على GitHub Pages"**
3. اضغط **Run workflow**

---

## ما الذي يحدث بالضبط؟

```
index.html على main
    ↓
GitHub Actions يشتغل
    ↓
Python يمسح GH_ENCRYPTED (يدعم var/let/const، علامات اقتباس مفردة ومزدوجة)
    ↓
التحقق الصارم: يفشل الـ build لو بقيت أي قيمة
    ↓
index.html النظيف → فرع gh-pages
    ↓
الطلاب يشوفوا الموقع ✅
```

## ملاحظات مهمة

| الموضوع | التفاصيل |
|---------|---------|
| 🔒 أمان الكود على main | الـ workflow بيحمي **الملف المنشور** فقط. لو حبيت تحمي الكود على main نفسه من أي بيانات حساسة، ارفع `index.html` بعد استخدام زر "📥 تصدير الموقع" داخل التطبيق (اللي بيمسح GH_ENCRYPTED قبل التنزيل). |
| 📱 رابط الموقع | `https://USERNAME.github.io/REPO-NAME/` |
| 🔄 التوافق مع زر النشر | زر "🚀 نشر على GitHub" في التطبيق **لا يزال يعمل** — الـ workflow مستقل عنه تماماً |
| ⚠️ مهم | الـ workflow ينشر `index.html` **فقط** — لو عندك صور أو ملفات تانية محتاجها، راسلني وأضيفها |
